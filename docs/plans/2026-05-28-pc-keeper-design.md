# PC Keeper — дизайн терминального приложения обслуживания Windows

Дата: 2026-05-28

## Контекст

Существующий проект `program_update_all` — PowerShell-инструмент агрегации обновлений Windows (winget, npm, choco, scoop, GitHub-Electron, local-git, драйверы). Цель — развить его в полноценное терминальное приложение «PC Keeper» с четырьмя разделами:

1. **Обновления** (существует) — обновление установленных программ.
2. **Ревизия ПК** — read-only отчёт о состоянии системы.
3. **Очистка** — поиск и удаление мусора.
4. **Безопасность** — гигиена настроек Windows + аудит учёток/доступа, с возможностью применять исправления.

Ограничение: только PowerShell-скрипты, без бинарников и внешних модулей.

## Принятые решения

- **Точка входа:** единое главное TUI-меню, переиспользующее существующий `Invoke-ChecklistMenu`.
- **Модель действий (гибрид):** ревизия — read-only; очистка реально удаляет (через чеклист + подтверждение); безопасность и показывает находки, и может применять fix (admin, dry-run по умолчанию).
- **UI:** чистый ANSI-рендер, ноль внешних зависимостей — сохраняет принцип «запускается одной командой где угодно».
- **Структура кода:** разбиение на дот-сорсируемые файлы по столпам + общие примитивы.
- **Рантайм:** pwsh 7, терминал с ANSI (Windows Terminal).

## 1. Архитектура и файлы

Корневой `ProgramUpdateAll.psm1` становится тонким: дот-сорсит части, держит `Export-ModuleMember` и точку входа `Invoke-PcKeeper` + `Invoke-MainMenu`.

```
src/
  ProgramUpdateAll.psm1   # дот-сорс частей, экспорт, точка входа
  Common.ps1              # Invoke-NativeText, Test-CommandAvailable, нормализация, версии
  Ui.ps1                  # ANSI-рендер: рамки, цвета, статусы, спиннер, прогресс, меню
  Update.ps1              # существующие провайдеры обновлений
  Audit.ps1               # железо/ОС, диски, автозагрузка, софт (read-only)
  Cleanup.ps1             # temp, кэши браузеров, анализ больших файлов
  Security.ps1            # гигиена БОС + аудит учёток/доступа
```

Принципы (адаптированные из Karpathy CLAUDE.md): каждый файл обозрим (<700 строк); UI отделён от логики — сканеры возвращают объекты, `Ui.ps1` рисует; паттерн «чистый парсер/сканер + тонкая обёртка + тест» во всех столпах.

Точки входа `.cmd`/`.ps1`: без флагов → `Invoke-MainMenu`; существующие флаги (`-List`, `-Doctor`, `-Inventory`, `-DryRun`, `-All`, `-SkipDrivers`, `-Yes`) остаются для прямого доступа и скриптинга.

## 2. UI-слой (`Ui.ps1`) и навигация

Чистые ANSI escape-коды (`[char]27`). Единый стиль:

- Статусы: `Ok` (✓ зелёный), `Warn` (⚠ жёлтый), `Bad` (✗ красный), `Info` (• серый).
- `Write-StatusLine -Status -Label -Value` — единый формат строк.
- `Write-Header -Title -Subtitle` — рамка box-drawing.

Функции:
- `Show-Banner` — заголовок + карточка «здоровье системы» (агрегат по 4 столпам).
- `Invoke-MainMenu` — цикл `Clear-Host` → рендер пунктов с цветом и сводкой → `ReadKey('NoEcho,IncludeKeyDown')`; `↑↓` навигация, `Enter` вглубь, `R` пересканировать, `Q/Esc` выход.
- `Invoke-ChecklistMenu` — переиспользуем, добавив цвет статусов и колонку размера/важности.
- `Show-Spinner` — крутится, пока выполняется scriptblock медленного сканера (winget, GitHub API).

Навигация двухуровневая: меню → экран столпа → `Esc` назад. Без глубокой вложенности (YAGNI).

UI-функции не добывают данные — сканеры (`Get-AuditReport`, `Get-CleanupTargets`, `Get-SecurityFindings`) возвращают объекты, меню только рисует. `ReadKey`/`Clear-Host` в тестах не вызываются.

## 3. Раздел «Ревизия ПК» (`Audit.ps1`, read-only)

`Get-AuditReport` → объект с четырьмя секциями, каждую собирает сканер через CIM-классы (без парсинга текста → без mojibake):

- **Железо и ОС** (`Get-HardwareInfo`): `Win32_Processor`, `Win32_PhysicalMemory`, `Win32_VideoController`, `Win32_BaseBoard`, `Win32_BIOS`, `Win32_OperatingSystem` (версия, билд, uptime).
- **Диски и место** (`Get-DiskInfo`): `Win32_LogicalDisk`, SMART через `Get-PhysicalDisk`/`HealthStatus`, топ папок по размеру (общая логика с очисткой).
- **Автозагрузка и службы** (`Get-StartupInfo`): `Win32_StartupCommand` + реестр `Run`, `Get-ScheduledTask`, `Get-Service` (автозапуск/остановленные).
- **Установленный софт**: переиспользуем существующий `Get-ProgramInventory`.

Форматтеры — чистые тестируемые функции (`Format-DiskLine`, `Format-Uptime`, `ConvertTo-HumanSize`). Диск >90% подсвечивается ⚠. Опционально экспорт в `%TEMP%\pc-keeper\audit-YYYYMMDD.txt`. Без admin не падает — показывает «нужны права для полного отчёта».

## 4. Раздел «Очистка» (`Cleanup.ps1`)

Поток как у обновлений: сканер → кандидаты (`New-CleanupCandidate`: `Category/Name/Path(s)/SizeBytes/RiskLevel/RequiresAdmin/RequiresClosedApp`) → чеклист → подтверждение → удаление.

Категории:
- **Безопасный temp-мусор** (`Get-TempCleanupTargets`): `%TEMP%`, `C:\Windows\Temp`, Корзина (`Clear-RecycleBin`), кэши npm/pnpm/choco/pip, старые `.log`.
- **Кэши браузеров** (`Get-BrowserCacheTargets`): только `Cache`/`Code Cache`/`GPUCache` Chrome/Edge/Firefox; `RequiresClosedApp` (предупреждение «закройте браузер»). Cookies/сессии/история не трогаем.
- **Анализ больших/старых файлов** (`Get-LargeOldFiles`): read-only, `RiskLevel = Review`; показываются, но не входят в «отметить всё», удаление каждого — отдельным явным выбором.

Предохранители:
- Удаление только путей из **белого списка** (`Test-IsSafeCleanupPath`); вне него — warning, пропуск. Никаких рекурсивных удалений по пользовательскому вводу.
- Сначала расчёт размера («освободится X ГБ»), удаление — `Invoke-CleanupCandidate -DryRun` по умолчанию + финальное подтверждение суммы.
- Ошибки по файлам собираются в `$failures`, не валят прогон.

## 5. Раздел «Безопасность» (`Security.ps1`)

`Get-SecurityFindings` → находки (`New-SecurityFinding`: `Check/Status/Title/Detail/FixCommand/FixArguments/RequiresAdmin/Reversible`).

**Гигиена БОС** (`Get-HygieneFindings`): Defender (`Get-MpComputerStatus`), брандмауэр (`Get-NetFirewallProfile`), BitLocker (`Get-BitLockerVolume`), UAC (реестр `EnableLUA`), SMBv1 (`Get-SmbServerConfiguration`), ожидающие обновления (переиспользуем сканеры обновлений), слушающие порты (`Get-NetTCPConnection -State Listen`, только показ), подозрительная автозагрузка (пересечение с ревизией).

**Аудит учёток** (`Get-AccountFindings`): члены «Администраторы» (`Get-LocalGroupMember`), учётки без требования пароля / «срок не истекает» (`Get-LocalUser`), включённый Guest, права на критичные папки (`Get-Acl`). Реальные пароли не читаются — только флаги/политика.

**Исправления** (`Invoke-SecurityFix`): только находки с заданным `FixCommand` (отключить SMBv1, включить брандмауэр, выключить Guest). Всегда `RequiresAdmin` → фильтр через `Get-PrivilegeBlockedUpdateCandidates`-паттерн. `-DryRun` по умолчанию + подтверждение по каждому + отметка `Reversible`. Без массовых авто-фиксов.

UI: «оценка здоровья» (`Get-SecurityScore` — чистая тестируемая), цветные строки, `[F]` исправить выбранное.

## 6. Тестирование, безопасность, фазы

**Тестирование** (расширяем `tests/run-tests.ps1`, паттерн `It` + `Assert-Equal`): чистые функции кормятся мок-объектами без обращения к системе. Обязательно покрыть: `ConvertTo-HumanSize`, `Test-IsSafeCleanupPath`, расчёт суммы выбранного, `Get-SecurityScore`, классификацию статусов гигиены, `Format-*`. Интерактив не тестируется. Набор зелёный до и после каждого шага.

**Сквозные предохранители:** удаление — только белый список; fix и системная очистка — dry-run по умолчанию + подтверждение + фильтр admin; ошибки в `$failures`; никаких рекурсий по произвольному вводу и авто-фиксов.

**Фазы (каждая = рабочее приложение, тесты зелёные):**
1. Рефакторинг-каркас — `Common.ps1`/`Ui.ps1`, дот-сорс, перенос обновлений в `Update.ps1`. 21 существующий тест проходит, поведение не меняется.
2. Главное меню (ANSI) — `Invoke-MainMenu`, баннер, навигация; обновления через меню.
3. Ревизия ПК — `Audit.ps1` + тесты форматтеров.
4. Очистка — `Cleanup.ps1` + тесты белого списка/размеров.
5. Безопасность — `Security.ps1` + тесты счёта/классификации; fix-команды последними.
6. Полировка UI — спиннеры, «оценка здоровья», экспорт отчётов.

**Допущения:** только PowerShell; pwsh 7; ANSI-терминал; без внешних модулей.
